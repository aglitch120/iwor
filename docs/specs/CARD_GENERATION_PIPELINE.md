# 国試合格カード生成パイプライン

> **目的**: 医師国家試験の過去問を入力として、「この科目のカードを完璧にすれば合格できる」フラッシュカードデッキを生成する。
>
> **実績**: 循環器領域で設計・検証済み（v1→v2の失敗と修正を経て確立）。

---

## 失敗から学んだ原則

v1（失敗）とv2（成功）の比較から導かれた、絶対に守るべき原則。

### ❌ やってはいけないこと（v1の失敗）

```
× 「〜を述べよ」「〜の定義は？」→ 教科書の暗記カード
× 知識の網羅（概念を均等に並べる）
× 心電図・画像所見をテキストで説明するだけ
× 禁忌肢を他のカードに埋め込む
× 鑑別を独立カードにしない
```

### ✅ やるべきこと（v2の成功パターン）

```
✓ frontは症例提示 or 臨床判断を問う形式
✓ 出題パターンの出現頻度に比例してカード枚数を配分
✓ 禁忌肢は独立カテゴリとして最優先
✓ 鑑別は「A vs B」の直接対比
✓ 心電図は「この波形→この疾患→この対応」の3点セット
```

---

## パイプライン全体像

```
[入力]                  [処理]                      [出力]
過去問データ ──→ Step 1: 出題パターン分析 ──→ パターン分類表
テキストPDF  ──→ Step 2: 禁忌肢抽出      ──→ 禁忌リスト
ガイドライン ──→ Step 3: カード生成       ──→ 6カテゴリのカード群
             ──→ Step 4: 品質検証       ──→ 最終デッキ
```

## Step 0: 入力データの準備

### 必要な入力

| 入力 | 形式 | 用途 |
|------|------|------|
| 過去問リスト | テキスト（問題番号＋疾患タグ＋問題文要約） | パターン分析の素材 |
| テキストPDF | PDF（medu4等） | 科目の全体構造把握 |
| 最新ガイドライン | URL or PDF | 数値・推奨の正確性担保 |

### 過去問データの取得方法

**推奨ソース**: 医療美術部の過去問チャート（https://medical-illustration.club/kakomon-chart/）
- 科目別・疾患別でソートされた過去問リスト
- 問題番号、疾患タグ、問題文の要約が含まれる
- web_fetchで取得可能

**対象範囲**: 直近10回分（110-119回）が目安。それ以前は出題傾向が変わっている可能性がある。

### テキストPDFの読み方

PDFはフォント問題で直接テキスト抽出できないことが多い（medu4等）。

```
1. pdfinfo で総ページ数確認
2. pdftotext で試しに抽出 → 文字化けならラスタライズ
3. pdftoppm で目次ページ（通常p1-5）をラスタライズ → 章構成を把握
4. 全ページ読む必要はない。章構成（＝科目のサブカテゴリ）の把握が目的
```

## Step 1: 出題パターン分析

### 目的

過去問を「何を判断させる問題か」で分類し、カード設計の骨格を作る。

### 7つの出題パターン

過去問を1問ずつ以下の7パターンに分類する。

| パターン | 定義 | カードカテゴリ |
|---------|------|--------------|
| **症例→診断** | 症例提示から疾患名を答えさせる | 🏥 症例→診断 |
| **症例→検査/画像→診断** | 心電図・CT等の所見判読を経由して診断 | 📊 パターン認識 |
| **症例→次の一手** | 初期対応・治療選択を問う | ⚡ 次の一手 |
| **鑑別診断** | 紛らわしい2疾患の区別 | 🔀 鑑別 |
| **禁忌肢** | やってはいけないことを問う | 🚫 禁忌 |
| **検査の選択** | どの検査を選ぶか | ⚡ 次の一手に統合 |
| **薬剤の選択/管理** | 薬剤の使用判断 | 💊 薬剤判断 |

### 分類プロンプト

過去問リストを与えて、以下のプロンプトで分類する:

```
以下の過去問リストを、出題パターンごとに分類してください。

出題パターン:
1. 症例→診断: 症例提示→疾患名を問う
2. 症例→検査/画像→診断: 心電図やCT所見の判読を経由して診断
3. 症例→次の一手: 初期対応・治療選択を問う
4. 鑑別診断: 紛らわしい2疾患の区別を問う
5. 禁忌肢: やってはいけない治療・薬剤を問う（誤っているものを選べ、含む）
6. 検査の選択: 次に行うべき検査を問う
7. 薬剤の選択/管理: 薬剤の使用判断を問う

各問題について:
- パターン番号
- 問われている核心（1文で）
- この問題を解くために必要な知識（1-2文で）

を出力してください。

[ここに過去問リストを貼る]
```

### カード枚数の配分ルール

パターン分析の結果に基づき、出題頻度に比例してカード枚数を配分する。

```
目安（1科目あたり50-70枚）:

🚫 禁忌肢:       全数カード化（通常8-15枚）← 最優先。1枚も漏らさない
🏥 症例→診断:     出題数の多い順に15-20枚
⚡ 次の一手:      10-15枚（検査選択を含む）
📊 パターン認識:   心電図・画像パターンとして8-12枚
🔀 鑑別:         紛らわしいペアを6-10枚
💊 薬剤判断:      5-8枚
```

**重要**: 出題されていない疾患のカードは作らない。出題0回の知識は国試に出ない。

## Step 2: 禁忌肢の網羅的抽出

### 目的

禁忌肢は3問踏むと即不合格。全科目横断で管理する最重要カテゴリ。

### 禁忌肢の抽出プロンプト

```
以下の科目の過去問データとテキスト内容から、
「やったら患者が死ぬ or 重篤な害をもたらす」行為を全て抽出してください。

抽出基準:
- 薬剤の禁忌（この疾患にこの薬→致死的）
- 処置の禁忌（この状態にこの手技→致死的）
- 見逃しの禁忌（この症状を放置→死亡）
- 検査の禁忌（この状態でこの検査→危険）

各禁忌について:
- 状況（どんな患者に）
- 禁忌行為（何をすると）
- 結果（何が起こるか）
- 正しい対応（代わりに何をすべきか）

を出力してください。過去問で直接出題されたものだけでなく、
出題されうる禁忌も含めてください。

[科目の過去問データとテキスト構成を貼る]
```

### 禁忌カードのテンプレート

```typescript
{
  id: "xx-kinki-NNN",
  front: "🚫 [状況の記述]。[禁忌行為]をした。何が起こるか？",
  back: "[結果の記述]\n\n【禁忌薬/禁忌行為】○○\n【正しい対応】△△",
  explanation: "[機序の説明。なぜ禁忌なのか。国試での出題パターン。]",
  tag: "🚫禁忌",
  difficulty: "intermediate"
}
```

## Step 3: カード生成

### 6カテゴリごとの生成プロンプト

#### 🚫 禁忌肢カード

Step 2の出力をそのままカード化。テンプレートは上記参照。

#### 🏥 症例→診断カード

```
以下の「症例→診断」パターンの過去問群から、フラッシュカードを生成してください。

ルール:
- front: 国試の症例提示に近い形式（年齢・性別・主訴・経過・所見を含む）
  ただし、元の問題文を絶対に引用しない。同じ概念を問う独自の症例を作る。
- back: 診断名（1行目）＋ 次のステップ（2行目）
- explanation: この症例の「決め手」（3つ以内）＋ 鑑別すべき疾患 ＋ 国試での出題傾向
- tag: "症例→診断"

品質基準:
- frontを読んだだけで診断に到達できる情報が含まれていること
- 「決め手」が明確であること（何がこの疾患を示唆するか）
- 解説に「国試ではこう問われる」パターンを含むこと

[ここに症例→診断パターンの過去問リストを貼る]
```

#### ⚡ 次の一手カード

```
以下の「次の一手」パターンの過去問群から、フラッシュカードを生成してください。

ルール:
- front: 臨床状況の提示 → 「まず行うべき対応は？」「次にすべき検査は？」
- back: 具体的なアクション（薬剤名・手技名を明記）
- explanation: なぜそのアクションが最優先か。他の選択肢がダメな理由。
- tag: "次の一手"

重要: 「検査の選択」問題もこのカテゴリに統合する。
「まず行うべき検査」も「次の一手」の一種。

[ここに次の一手パターンの過去問リストを貼る]
```

#### 📊 パターン認識カード（心電図・画像・身体所見）

```
以下の「検査/画像→診断」パターンの過去問群から、パターン認識カードを生成してください。

ルール:
- front: 📊 接頭辞 ＋ 所見の記述（波形の特徴をテキストで） → 「診断と対応は？」
- back: 診断名 ＋ 即座に必要なアクション
- explanation: 所見の解剖学的/生理学的意味 ＋ 類似パターンとの鑑別
- tag: "心電図" or "画像" or "身体所見"

注意:
- 心電図は「この波形→この疾患」の直結を重視。機序は解説に回す
- 実際の画像は表示できないため、テキストでの波形記述を可能な限り具体的に
  例: 「V1-V3でcoved型ST上昇＋陰性T波」→ Brugada
- 聴診所見も同様: 「心尖部の汎収縮期雑音、左腋窩放散」→ MR

[ここにパターン認識の過去問リストを貼る]
```

#### 🔀 鑑別カード

```
以下の過去問分析から、紛らわしい疾患ペアを特定し、鑑別カードを生成してください。

ルール:
- front: 【鑑別】A vs B — 決定的な違いは？（or 特定の臨床状況での区別点は？）
- back: 鑑別ポイントを2-3項目、対比形式で
- explanation: なぜ紛らわしいのか ＋ 国試でどう問われるか
- tag: "鑑別"

鑑別ペアの見つけ方:
- 同じ症状で来院するが治療が異なる疾患（例: ASの胸痛 vs 狭心症の胸痛）
- 同じ検査所見だが原因が異なる疾患（例: ST上昇: STEMI vs 心膜炎 vs 冠攣縮）
- 同じ身体所見だが機序が異なる疾患（例: 壁肥厚: HCM vs アミロイドーシス）
- 類似した名前/概念で混同しやすいもの

[ここに過去問リストと分析結果を貼る]
```

#### 💊 薬剤判断カード

```
以下の過去問から、薬剤の使用判断に関するカードを生成してください。

ルール:
- front: 💊 接頭辞 ＋ 臨床状況 → 「使うべき薬剤は？」or「この処方の問題点は？」
- back: 薬剤名 ＋ 根拠
- explanation: 作用機序（簡潔に） ＋ エビデンス/ガイドライン ＋ 禁忌との区別
- tag: "薬剤判断"

注意:
- 禁忌肢と重複する内容は禁忌カード側を優先し、薬剤カードでは
  「正しい選択」側をカード化する
- 最新ガイドラインの数値（用量等）はweb検索で確認してから記載

[ここに薬剤関連の過去問リストを貼る]
```

## Step 4: 品質検証

### チェックリスト（全カード共通）

```
□ frontを読んだだけで回答に到達するための情報が十分か？
□ backは明確で曖昧さがないか？（「〜が考えられる」ではなく断定）
□ 過去問の問題文・選択肢を引用していないか？（著作権）
□ テキストの文章をそのままコピーしていないか？
□ ガイドラインの数値（用量・基準値）は最新版で確認済みか？
□ explanationに「国試でどう問われるか」のパターンが含まれているか？
□ ID重複がないか？
```

### 合格シミュレーション検証

最終チェックとして、以下を自問する:

```
このデッキの全カードを完璧に理解・暗記した学生が、
直近3回の国試（117-119回）のこの科目の問題を解いたら、
何割正答できるか？

目標: 85%以上

85%未満の場合:
- カバーされていない出題パターンがある → カード追加
- カードはあるが解けない問題がある → カードの問い方を修正
- 画像問題で解けない → パターン認識カードの記述を改善
```

### 科目固有の注意点チェック

```
□ 最新ガイドライン変更点を反映しているか？
  （例: 心不全GL 2025改訂のSGLT2i推奨、PAHのmPAP基準変更）
□ 日本固有のプラクティスを反映しているか？
  （例: H.pylori除菌レジメン、降圧目標値は海外GLと異なる）
□ 禁忌肢は全て独立カード化されているか？（他カードに埋没していないか）
```

## Step 5: TypeScriptファイル出力

### ファイル構成

```typescript
// ファイル名: {科目名}-cards.ts
// 配置先: app/study/data/

export interface FlashCard {
  id: string        // "{科目略称}-{カテゴリ}-{連番}" e.g. "jk-kinki-001"
  front: string     // 問い（症例提示 or 判断を問う）
  back: string      // 回答（診断名＋アクション）
  explanation: string // 解説（機序＋国試出題パターン）
  tag: string       // カテゴリ名
  source?: string   // ガイドライン参照（あれば）
  difficulty?: 'basic' | 'intermediate' | 'advanced'
}
```

### ID命名規則

```
{科目2文字}-{カテゴリ}-{3桁連番}

科目略称:
  jk = 循環器, kk = 呼吸器, sh = 消化器, ns = 神経, 
  nf = 腎泌尿器, nd = 内分泌, kt = 血液, kn = 感染症,
  mn = 免疫, ss = 整形, sn = 産婦人科, sh = 小児科,
  se = 精神, hf = 皮膚, gn = 眼科, jb = 耳鼻科,
  ky = 救急, hs = 必修, kw = 公衆衛生

カテゴリ:
  kinki = 禁忌, dx = 診断, tx = 治療/次の一手,
  ecg = 心電図, img = 画像, ddx = 鑑別, drug = 薬剤
```

### メタデータ

```typescript
export const deck_metadata = {
  name: string,           // "循環器 v2（国試逆算設計）"
  version: string,        // "2.0.0"
  totalCards: number,
  structure: [
    { name: string, count: number, priority: string }
  ],
  examCoverage: string,   // "第110-119回 循環器94問"
  guidelineReferences: string[],
  generatedAt: string,    // ISO日時
}
```

## 運用: Claude.ai / Claude Code での使い方

### Claude.ai（このチャット）での使い方

```
1. 過去問チャートのURLを貼る（or 過去問リストをテキストで渡す）
2. テキストPDFをアップロード
3. 「このパイプラインで○○科のカードを作って」と指示

→ Claude.aiがStep 1-5を実行し、.tsファイルを出力
```

### Claude Code での使い方

```
1. CLAUDE.md or docs/ にこのパイプラインを配置
2. サブエージェント card-generator を作成（.claude/agents/card-generator.md）
3. 過去問データを渡して実行

→ app/study/data/{科目}-cards.ts に直接出力＋コミット
```

### 段階的な品質改善

```
初回生成 → ユーザー（医師/医学生）レビュー → フィードバック反映 → v2
                                                               ↓
                                                     次の国試結果で検証
                                                               ↓
```
