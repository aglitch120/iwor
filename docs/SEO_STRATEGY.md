## 1. ゴールドスタンダード定義

### SaaSスタートアップのSEOベストプラクティス（業界標準）

| 原則 | 説明 | iwor適用 |
|------|------|---------|
| **Topic Cluster Model** | ピラーページ + クラスター記事 + 内部リンク | 各クラスターにピラーページ作成 |
| **Search Intent Matching** | TOFU/MOFU/BOFU別にコンテンツ設計 | BOFU（J-OSLER/試験対策）を最優先 |
| **Programmatic SEO** | テンプレート×データで大量ページ自動生成 | 166計算ツールページ（既存）+ 病院DB 1,470ページ |
| **Content Velocity** | 量×質×更新頻度 | 月8-12本新規 + 既存記事の定期更新 |
| **Technical SEO** | Core Web Vitals、構造化データ、サイトマップ | Cloudflare Pages（高速）+ JSON-LD対応 |
| **Internal Linking** | 全ページが3クリック以内で到達可能 | 孤立ページ解消、関連記事自動推薦 |
| **E-E-A-T** | Experience/Expertise/Authoritativeness/Trustworthiness | 医師が作成・監修していることを明示 |
| **YMYL対策** | Your Money Your Life領域は特に信頼性が重要 | 医療情報は根拠明示、免責事項、著者プロフィール |

### 日本の医療IT特有のSEO事情

- **YMYL（Your Money or Your Life）**: 医療は最もE-E-A-Tが重視される領域
- **Google日本語検索の特性**: 長尾キーワードが英語圏より重要（「J-OSLER 病歴要約 書き方 テンプレート」等）
- **競合のSEO成熟度が低い**: 医師ブロガーは個人運営→SEO知識が浅い。iworが体系的にやれば勝てる
- **検索ボリュームは小さいが転換率が極めて高い**: 「J-OSLER 病歴要約 書き方」で検索する人は100%内科専攻医→ペルソナ完全一致

## 9. iwor独自のSEO優位性

1. **Programmatic SEO**: 166計算ツール + 1,470病院DB = 1,636ページが自動生成
2. **Tool-as-Content**: 計算ツール自体がコンテンツ（被リンク獲得しやすい）
3. **コンテンツの実用性**: 読んで終わりではなくツールを使える（滞在時間↑）
4. **更新頻度**: 計算ツールのガイドライン更新を追跡→freshness signal
5. **ニッチ支配**: 「J-OSLER」関連KWで既に66記事→圧倒的カバレッジ
