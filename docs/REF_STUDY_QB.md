## 1. コンセプト

**「QB＋Anki＋AI家庭教師」の統合体を、基本無料で提供する**

- 全過去問演習: 無料（ユーザー獲得ファネル）
- AI解説・カード生成・苦手分析: PRO（クレジット課金）
- medu4終了＋国試CBT化のタイミングで市場の空白を取る

## 2. 法務リスク評価（確定）

| 要素 | リスク | 根拠 |
|---|---|---|
| 問題文＋選択肢＋正答の公開 | 極めて低い | 厚労省が自らPDF公開。medu4/QB/MEC等が商用利用、厚労省から権利主張事例ゼロ |
| AI生成の解説 | ゼロ | iwor studyのオリジナル著作物 |
| 公開画像の利用 | 低い | 厚労省公開分のみ使用 |
| 科目分類・タグ付け（自前） | ゼロ | 独自体系 |
| 出典表記 | 必須 | 「厚生労働省 第XXX回医師国家試験より」を全問に表記 |

**medu4からのデータ取得方針:**
- 解説文: 取得しない
- 正答率: 取得しない（自前で蓄積）
- 科目・テーマ・形式: 検索結果一覧を手動コピペ（著作権リスク極小）
- 問題文: medu4からではなく厚労省PDFから取得

**mediLink利用規約 第7条(3):** 複製・二次著作物作成を明示的に禁止。QBのスクショ取り込み等の機能は作らない。

## 3. データ構築パイプライン（確定）

### 3.1 ソースとフロー

```
[厚労省PDF] ──テキスト抽出（deterministic）──→ 問題文・選択肢
[厚労省PDF] ──別PDF──→ 正答
[medu4検索結果] ──手動コピペ──→ 科目・テーマ・形式・正答率
                                    ↓
                              統合JSON/DB
                                    ↓
                         Claude APIで分類タグ付与（iwor独自体系）
                                    ↓
                         関連問題リンクをバッチ生成
```

### 3.2 厚労省PDFの特性

- デジタルテキスト埋め込みPDF（OCR不要）
- 問題本文のテキスト抽出: pdfplumber / pymupdf で可能（実証済み: 119A全75問を抽出成功）
- 別冊（画像問題）: 画像として抽出。一部は厚労省が非公表/加工済み
- 正答: 別PDFで公開。テーブル形式、パース容易
- 対象回数: 100回〜120回（medu4の収録範囲と一致）

### 3.3 ハルシネーション対策

- Phase 0（テキスト抽出）: AIを使わない → ハルシネーションゼロ
- Phase 0.5（分類タグ付与）: AIだが、分類の誤りは致命的でない（後から修正可能）
- Phase 1（解説生成）: 正答は厚労省データ（AIが判断しない）。解説のみAI生成
- 3層防御: ①正答は公式データ ②静的解説は医師レビュー ③対話はRAG＋免責表示

## 4. 分類体系（仮説 — Phase 0.5でボトムアップ再設計）

### 4.1 臨床領域（Clinical Domain）— 20分類

medu4の27分類と意図的に番号体系をずらす（アルファベット＋数字）。

```
A. 内科系メジャー
  A1 循環器・血管  A2 呼吸器  A3 消化管  A4 肝胆膵
  A5 腎・電解質  A6 内分泌・代謝・栄養  A7 血液・腫瘍
  A8 免疫・アレルギー・膠原病  A9 感染症・微生物  A10 神経・筋
B. 外科系・専門科
  B1 整形外科  B2 眼科  B3 耳鼻咽喉科  B4 泌尿器科  B5 皮膚科  B6 精神科
C. 周産期・小児
  C1 産科・婦人科  C2 小児科・新生児
D. 横断領域
  D1 救急・集中治療・中毒・麻酔  D2 放射線・画像・病理
  D3 公衆衛生・社会医学・医療倫理  D4 総論（輸液・栄養・周術期・緩和・医学総論）
```

### 4.2 タグ体系（仮説 — Phase 0.5で全問データから帰納的に導出）

- 臨床アクションタグ: 暫定6種（診断/検査選択/治療選択/病態理解/予防/対応・説明）→ 全問を見て再設計
- 知識タイプタグ: 暫定5種（事実/機序/臨床判断/計算/英語）→ 全問を見て再設計
- カードタイプ: 暫定5種（事実/機序/鑑別/臨床判断/ゴロ）→ 全問を見て再設計

**重要: これらは直感ベースの仮説。Phase 0.5で8,000問をClaude APIに「自由記述で」分類させ、クラスタリングして実データに基づく体系を構築する。**

## 5. AI解説生成（確定）

### 5.1 モデル使い分け

| 処理 | モデル | コスト |
|---|---|---|
| 静的解説（事前生成、1回） | Opus 4 | ~24,000円/全8,000問 |
| 対話型解説 | Sonnet 4 | クレジット消費 |
| カード生成 | Haiku 4.5 | クレジット消費 |
| 苦手分析レポート | Sonnet 4 | クレジット消費 |
| 関連問題リンク | バッチ処理済み | API不要 |

### 5.2 プロンプト設計の要点

- 正答はAIが判断しない（厚労省データを注入）
- 各選択肢に○/×＋理由を付与
- 背景知識はイヤーノートレベルの体系的まとめ
- 対話時はユーザーの演習データ（苦手領域・正答率）をsystem promptに注入
- カード生成は解説から「覚えるべき知識」を抽出→複数カードに分解

→ プロンプト全文は iwor_study_design.md（ローカル設計書）参照

## 6. 課金モデル（確定）

### 6.1 クレジット制

```
無料: 月30クレジット（体験用）
PRO（月980円に含む）: 月300クレジット
追加購入: 100クレジット = 500円
直前パック: 1,000クレジット = 3,980円（季節商品）
国試フルパック: 3,000クレジット = 9,800円（季節商品）
```

### 6.2 クレジット消費表

| アクション | 消費 | 実APIコスト | 粗利率 |
|---|---|---|---|
| 問題演習（解く＋正答表示） | 0 | 0円 | — |
| 静的解説を読む | 0 | 0円 | — |
| AI対話1往復 | 3 | ~1.5円 | 75% |
| カード自動生成（1問分） | 2 | ~1円 | 75% |
| 深掘り対話（長文） | 5 | ~3円 | 70% |
| 苦手分析レポート | 10 | ~5円 | 75% |

**核心: 「解く」「読む」は永久無料。AIを「使う」ときだけクレジット消費。**

### 6.3 コスト最適化

- 全問の静的解説をDBキャッシュ → 2人目以降はAPIコスト0
- よくある質問パターンもKVキャッシュ
- Haiku活用でカード生成・簡単な質問はコスト1/10
- 月間APIコスト/ユーザー: 約150円（粗利率80%超）

## 7. 機能仕様（確定）

### 7.1 第1層: 過去問演習（無料）
- 全過去問DB（100回〜最新回、約8,000問）
- 科目別・タグ別フィルター・検索
- 問題を解く→正誤判定→静的解説表示
- 演習データ保存（ユーザー登録必須→ロックイン）
- 全問題ページのSSG（SEO用8,000ページ）
- 正答率表示（ユーザー蓄積データ）
- 出題頻度分析・トレンド表示

### 7.2 第2層: AI学習支援（PRO/クレジット）
- AI解説オンデマンド生成（各選択肢の正誤理由＋背景知識）
- 「なぜ？」対話（ユーザーの苦手データ注入済み）
- 暗記カード自動生成（問題→メタ認知→カード化）
- 忘却曲線復習（SM-2 / FSRS）
- 関連問題リンク（同じ疾患・概念の過去問を自動表示）

### 7.3 第3層: パーソナライズ（PRO）
- 苦手分析ダッシュボード（科目・疾患・知識タイプ別）
- 統計タブ（完了率: 回数別/分野別/問題タイプ別、時系列グラフ）
- AI学習コーチ（フリーチャット: 何を勉強すべきか、戦略相談）
- 進捗レポート（週次メール/通知）

### 7.4 ユーザー属性別モード
- 医学生（CBT対策）/ 医学生（国試対策）/ 初期研修医 / 専攻医 / 一般医師
- 属性によりトップ表示のQB、AIトーン、推奨学習パスが変わる
